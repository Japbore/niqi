## v0.1.0 — Versión Inicial

### Historias de Usuario

| ID     | Requisito                                                  |
|--------|------------------------------------------------------------|
| RNF-01 | Los datos se persisten en IndexedDB (no se pierden al cerrar) |
| RNF-02 | La UI es responsive y usable en móvil                      |

### Casos de Uso

#### CU-01: Añadir producto
1. El usuario escribe el nombre del producto.
2. Pulsa "Añadir" (o Enter).
3. El producto aparece en la lista con cantidad 1 y estado "pendiente".

#### CU-02: Marcar como comprado
1. El usuario pulsa sobre el checkbox de un producto pendiente.
2. El producto se mueve a la sección de "comprados" (tachado).

#### CU-03: Desmarcar producto
1. El usuario pulsa sobre el checkbox de un producto comprado.
2. El producto vuelve a la sección de pendientes.

#### CU-04: Eliminar producto
1. El usuario pulsa el botón de eliminar junto a un producto.
2. El producto desaparece de la lista.

### Casos de Uso

#### CU-05: Usar offline
1. El usuario abre la app con conexión (primera vez).
2. El Service Worker cachea los recursos.
3. El usuario pierde conexión.
4. La app sigue funcionando normalmente.

#### CU-06: Instalar en móvil
1. El usuario abre la app en Chrome/Safari.
2. Aparece opción "Añadir a pantalla de inicio" (o el usuario la busca en el menú).
3. La app se instala y aparece como icono en el móvil.

#### CU-07: Asignar categoría a un producto
1. El usuario escribe el nombre del producto.
2. Selecciona una categoría del desplegable (opcional).
3. Pulsa "Añadir".
4. El producto aparece en la lista, agrupado bajo su categoría.

### Importación Masiva

### Historias de Usuario

| ID    | Como...             | Quiero...                                          | Para...                                  |
|-------|---------------------|----------------------------------------------------|------------------------------------------|
| HU-09 | usuario             | importar una lista masiva de productos desde texto | ahorrar tiempo al configurar la lista    |

### Requisitos Funcionales

| ID    | Requisito                                                   | HU relacionada |
|-------|-------------------------------------------------------------|----------------|
| RF-11 | Botón de importación que abre un área de texto (textarea)   | HU-09          |
| RF-12 | Procesador de texto para importar (formato: `nombre,categoría`) | HU-09          |

### Casos de Uso

#### CU-08: Importar lista masiva desde texto
1. El usuario pulsa el botón de "Importar".
2. Se abre un área de texto (o modal).
3. El usuario pega o escribe el texto (ej: `Leche,Lácteos\nPan,Panadería\nHuevos`).
4. El sistema procesa cada línea:
   - Si tiene coma, divide en `nombre` y `categoría`.
   - Si no tiene coma, usa el texto completo como `nombre` y categoría vacía.
5. Los artículos se añaden a IndexedDB y se refresca la lista.
